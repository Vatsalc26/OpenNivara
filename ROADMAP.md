# OpenNivara Roadmap

This roadmap is intentionally stabilization-first. It describes current contributor direction without promising timelines.

## Currently Implemented

- Desktop app built with Tauri and React
- Rust CLI
- SQLite-backed sessions and memory
- Workspace map and controlled local tools
- Optional Telegram integration
- Marketplace themes
- Data-only skill packs

## Actively Upgrading

- Enforce memory privacy flags before prompt context reaches Gemini
- Harden desktop local-tool boundaries and finish explicit approval UX
- Improve first-run desktop setup and API-key guidance
- Keep skills activation inspectable and disabled by default
- Maintain CI coverage and smoke tests

## Future Ideas

- Local model provider options
- Secure desktop credential storage
- Expanded pack quality evaluation
- Better workspace setup UX
- Installer and release-readiness tooling
- Broader data-only plugin ecosystem

## Known Deferred Work

- Full approval workflow for medium/high-risk tools
- Canonical-path and symlink hardening
- Encrypted local database storage evaluation
