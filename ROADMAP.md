# OpenNivara Roadmap

This roadmap is intentionally stabilization-first. It describes current
contributor direction without promising timelines.

## Now

- Enforce memory privacy flags before prompt context reaches Gemini.
- Harden desktop local-tool boundaries and finish explicit approval UX.
- Improve first-run desktop setup and missing API-key guidance.
- Keep skills activation inspectable, data-only, and disabled by default.
- Maintain CI coverage for Rust, desktop frontend, and Tauri smoke checks.

## Next

- Add secure desktop credential storage for Gemini and Telegram secrets.
- Expand pack conflict handling and pack quality evaluation.
- Improve workspace map setup from desktop without terminal-only steps.
- Prepare installer builds and release-readiness checks.

## Later

- Add local model provider options.
- Evaluate encrypted local database storage.
- Broader India-relevant skills based on user feedback.
- Build a broader data-only plugin ecosystem without executable marketplace code.
