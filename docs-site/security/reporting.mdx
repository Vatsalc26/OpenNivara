Do not open public GitHub issues for sensitive vulnerabilities.

Read the repository [SECURITY.md](https://github.com/Vatsalc26/OpenNivara/blob/main/SECURITY.md) before reporting a vulnerability.

Current verified contact: send a direct message on X to [@choco_chip2m](https://x.com/choco_chip2m).

## Sensitive Issues

Sensitive issues include:

- API-key or Telegram bot-token disclosure.
- Telegram authorization bypass.
- Unauthorized local-file access.
- Exposure of profiles, preferences, memories, sessions, saved locations, prompts, or logs.
- Unsafe tool execution.
- Sensitive data included in generated artifacts.

## What To Include

Start with:

- a concise summary
- affected area
- safe reproduction outline
- impact assessment
- whether credentials, local files, memory, sessions, Telegram access, or logs are involved

Do not include secrets, raw credentials, `.env` files, private local databases, or full sensitive logs in an initial report.
