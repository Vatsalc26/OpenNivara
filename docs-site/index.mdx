OpenNivara is an alpha-stage, desktop-first personal AI agent. It combines a Rust backend, a Tauri + React desktop app, a Rust CLI named `opennivara`, SQLite/TOML local state, Gemini API integration, optional Telegram access, memory, skills, marketplace packs, visual themes, and controlled local tools.

OpenNivara is local-first, not offline-only. Your configuration, memories, sessions, workspace-map data, and runtime state are intended to live on your device by default, but selected context may be sent to Gemini. Telegram-based requests pass through Telegram.

## Core Surfaces

OpenNivara has two main user surfaces:

- Desktop app: the primary product surface for chat, settings, memory, store, skills, and desktop validation.
- CLI: a setup, diagnostics, chat, workspace-map, session, marketplace, skills, and Telegram surface.

The browser Vite app is useful for frontend iteration only. It is not the full desktop application and should not be used as proof that Rust-backed behavior works.

## Alpha Safety Notice

OpenNivara can process sensitive local data, including profile fields, preferences, sessions, memories, saved locations, selected file contents, prompts, tool arguments, Telegram metadata, and logs. Local state is not claimed to be encrypted.

Interactive approval enforcement for some local-tool actions is still under development. Canonical-path and symlink hardening is planned. Gemini API-key header transport is planned hardening. Treat this release as an early public alpha.
