---
title: "Feature Contract"
description: "Public-facing rules for preserving OpenNivara privacy, Store/Settings boundaries, and desktop validation when adding features."
---

New features should preserve these boundaries:

- Local-first state remains local by default, but selected context may be sent to Gemini.
- Desktop is the primary validation surface for Rust-backed behavior.
- Store themes are visual only.
- Store may preview and install data-only skill packs.
- Installing a skill pack must not activate prompt behavior.
- Settings -> Skills controls skill enablement and inspection.
- Skill packs must not include executable plugin runtimes, WASM runtimes, native scripts, remote skill execution, network tools, or arbitrary code execution.
- Tool behavior remains constrained by local configuration, remote policy, and backend skill/tool policy.

Do not claim encryption, complete approval enforcement, symlink escape protection, or Gemini header-based API-key transport until those are implemented and verified.
