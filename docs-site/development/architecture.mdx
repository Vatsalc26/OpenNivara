OpenNivara combines:

- Rust backend crate and CLI binary named `opennivara`.
- Tauri desktop application.
- React frontend.
- SQLite databases for sessions, memory, workspace-map data, graph-derived data, and audits.
- TOML configuration for profile, style, preferences, contexts, tools, Telegram, skills, marketplace, and modes.
- Gemini API integration for model responses.
- Optional Telegram daemon.
- Marketplace packs, visual themes, and data-only skill manifests.

## Source Layout

- `src/`: Rust backend, CLI, engine, memory, tools, skills, marketplace, sessions, Telegram, and workspace map.
- `desktop/`: Tauri + React desktop app.
- `packs/builtin/`: first-party built-in packs and themes.
- `docs/`: internal engineering documentation.
- `docs-site/`: public Mintlify documentation source.

Internal docs can be more implementation-oriented. Public docs should stay accurate, user-facing, and conservative about alpha limitations.
