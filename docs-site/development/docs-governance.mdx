---
title: "Docs Governance"
description: "How OpenNivara separates internal engineering docs from public Mintlify docs."
---

`docs/` is the internal canonical engineering tree. `docs-site/` is the public-facing Mintlify documentation source.

Public docs should:

- use user-facing prose
- avoid internal planning text unless summarized for users
- keep alpha limitations visible
- avoid unsupported security claims
- avoid unverified deployment URLs, analytics IDs, feedback integrations, or secrets
- distinguish current behavior from planned hardening

## Local Commands

```bash
bun run docs:check
bun run docs:site:check
cd docs-site
bun run dev
```

Do not commit Mintlify tokens, deployment credentials, or secrets.
